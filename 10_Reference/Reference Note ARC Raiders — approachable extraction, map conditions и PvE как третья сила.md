# ARC Raiders: approachable extraction, map conditions и PvE как третья сила

> Речь о ARC Raiders от Embark. Это reference для доступного extraction-порога, живых условий карты, яркого хаба и PvE-угрозы, которая заставляет игроков принимать решения без полного ухода в хардкорную непрозрачность.

## 1. Проверенная база источников

Официальный сайт ARC Raiders на 25.06.2026 описывает игру как multiplayer extraction adventure на смертельной будущей Земле, где игроки-рейдеры выходят на поверхность за ресурсами и сталкиваются с машинами ARC и другими рейдерами. На главной странице видны live map conditions, включая погодные и ночные модификаторы.

Steam-страница уточняет применимые механики: машины имеют разные сильные стороны и слабые места; шум боя слышен другим игрокам; выживание строится на крафте из найденных материалов, апгрейде workshop stations, permanent blueprints, skill tree из Survival/Mobility/Conditioning, оружии, гранатах, ловушках, зиплайнах, deployables и augments.

## 2. Культура дизайна ARC Raiders

### Extraction как приключение, а не экзамен на вики

ARC Raiders заметно мягче подаёт extraction-жанр:

- понятная фантазия: выйти наверх, добыть, вернуться в Speranza;
- красивый и читаемый мир вместо только грязного милсим-стресса;
- PvE-машины являются узнаваемой внешней угрозой;
- билд строится через навыки, крафт и гаджеты;
- игрок может играть соло, дуо или сквадом.

Для Eldraine это поддерживает уже записанный принцип `Scavenger Comfort`: хаб должен психологически разгружать, а рейд быть жестоким, но читаемым.

### Map Conditions как живой слой

ARC Raiders показывает на сайте активные условия карт. Это важная UX-идея:

```text
карта
  + текущее условие
  + временное окно
  + ожидаемый риск
  + особая добыча
= осознанный выбор рейда
```

Для Eldraine:

- Hub Map Table должен показывать не только `T1/T2/T3`, но и локальные модификаторы;
- условия должны менять маршруты, звук, видимость, тип лута и требования среды;
- игрок выбирает "какой риск сейчас интересен", а не просто "где больше награда".

Важно не повторить проблему фиксированных событий по неудобному расписанию. Rolling pool Eldraine уже лучше: всегда есть T1/T2/T3. Событийный слот D можно использовать для редких условий, но базовые фазовые возможности должны быть доступны без ожидания конкретного часа реального дня.

### PvE как третья сила

Машины ARC ценны не тем, что просто мешают игроку. Они:

- имеют разные профили поведения;
- вынуждают шуметь и раскрывать позицию;
- могут привлекать третьих игроков к уже начатому бою;
- создают выбор: обойти, потратить ресурс, устроить засаду, использовать угрозу против конкурента.

Для Eldraine это переводится в Аномалию и мобов:

- моб должен быть профилем поведения (`swarm`, `ambusher`, `armored`, `charger`, `caster`, `support_core`), а не просто HP-болванкой;
- шумный бой должен менять meso-состояние района;
- PvE не должно автоматически наказывать только активного игрока, иначе оно усилит кемпинг;
- лучшая версия PvE заставляет статичного игрока тоже платить: патруль, энтропия, деградация света, смена маршрута.

## 3. Что брать в Eldraine

### Гаджеты вместо универсальных способностей

ARC Raiders даёт глубину через granades, traps, ziplines, deployables и augments. Eldraine может использовать этот слой, чтобы не перегружать `P/Q/E`.

Применение:

- зиплайн/верёвка/крюк как предмет или карта, а не бесплатный геройский dash;
- ловушка как подготовленное committed-действие, требующее времени и руки;
- deployable как шумный контроль зоны, а не мгновенная стена;
- augment как предрейдовый выбор слота, закрывающий одну слабость.

### Skill Tree как осторожная мета, не как чистый power creep

ARC делит рост на Survival, Mobility и Conditioning. Для Eldraine аналог лучше делать через фракционные доктрины, трейты Пешек и сервисы Осколка.

Применимые оси:

- Survival: фильтры, медленное лечение, стабилизация, перенос травм;
- Mobility: маршруты, тишина, вертикальные окна, выход из перегруза;
- Conditioning: stamina, перенос веса, устойчивость к Heat/Backlash.

Запрет:

- не давать постоянный универсальный урон;
- не превращать аккаунт в неизбежного победителя новичков;
- не закрывать базовые слабости ячейки `Race x Spec`.

### Крафт как память маршрутов

Permanent blueprints и workshop stations хорошо подходят Eldraine:

```text
нашёл чертёж
  -> вынес
  -> стабилизировал
  -> открыл повторяемый рецепт
  -> следующий рейд получил новую цель обслуживания
```

Это усиливает loop "вылазка меняет будущие вылазки" без необходимости выдавать прямые уровни урона.

## 4. Что не брать

- слишком гладкую approachable-подачу, если она убирает страх потери Пешки;
- PvE, которое работает только как внешний тир;
- map events, запертые за неудобным реальным расписанием;
- гаджеты, которые заменяют биологию рас и оружейные доктрины;
- safe pocket как безусловную страховку ценных предметов без лорной цены.

## 5. Анти-кемпинг через ARC-линзу

Сильный урок ARC: шум боя привлекает других игроков. Но для Eldraine этого мало, потому что игрок может решить "тогда не шумлю".

Нужна пара сил:

| Поведение | Давление |
|---|---|
| активный бой | шум, Pulse, расход батарей, PvE/игроки слышат |
| полная статика | энтропия, холод, деградация укрытия, патруль, потеря окна лута |
| жадный лут | вес, DissonanceLoad, медленный выход |
| безопасное ожидание | нулевая прибыль, закрывающиеся цели, изменение маршрутов |

Иными словами:

```text
активность опасна
бездействие тоже опасно
решение игрока - выбрать форму опасности
```

## 6. Короткий вывод для Eldraine

Брать:

- видимые map conditions в Hub Map;
- PvE-профили как третью силу, а не только мясо;
- гаджеты и deployables как слой решений между оружием и Q/E;
- permanent blueprints как долгосрочную добычу;
- уютный, функциональный хаб как контраст рейду.

Прототипировать:

- как условия карты меняют маршруты и звук;
- сколько гаджетов можно дать, не убив `Race x Spec`;
- нужен ли ограниченный safe pocket для Trace/quest-item и какая у него лорная цена;
- как сделать PvE опасным для кемпера без спавна врага "из воздуха".
