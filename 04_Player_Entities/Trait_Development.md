# Chronicle: память, а не дерево перков

## 1. Обещание

Chronicle хранит пережитые факты конкретной смертной Пешки: возвращения, раны, спасённых людей, утраченные вещи, освоенные процедуры, отношения и невыполненные обещания. Запись объясняет, откуда взялось личное свойство и почему потеря человека имеет значение, но сама по себе не является механическим тегом.

```text
Chronicle entry != personal_tag != quest
```

- Chronicle отвечает: **что произошло с этим человеком**.
- Personal Tag отвечает: **какое устойчивое механическое свойство теперь принадлежит Пешке**.
- Quest Engine отвечает: **какая цель, сторона и мировое последствие продолжают происшествие**.

Chronicle не имеет общего лимита. Механические Personal Tags ограничены тремя по [[04_Player_Entities/Tags_System|системе тегов]].

## 2. Как событие становится тегом

Не каждое значимое решение должно выдавать perk. Событие создаёт тег только тогда, когда оставило проверяемое свойство тела, освоенного действия или физического носителя.

```text
visible event
  -> Chronicle records the fact
  -> eligible bodily / practiced / carrier outcome exists
  -> one explicit Personal Tag may be attached
```

Примеры границы:

| Произошедшее | Chronicle | Personal Tag |
|---|---|---|
| Пешка бросила дорогой груз, чтобы вынести человека | записывает выбор и последствия | не возникает автоматически |
| Пешка пережила воздействие, изменившее её кожу или обмен | записывает источник и лечение | допустима mutation/scar-запись |
| Пешка освоила конкретную фазу уже разрешённого Frame | записывает практику и свидетеля | допустим mastery-тег |
| Спасённый Foundling ищет старый адрес | хранит происхождение | цель исполняет Origin Continuation через Quest Engine |

`Trouble -> Leverage -> Residue` остаётся допустимой грамматикой персональной ситуации или квестового продолжения. Она не является шаблоном механического тега и не превращает цельную способность либо поручение в пассивное свойство.

## 3. First Return Ward

Первая успешная вылазка Ward всегда создаёт запись `First Return` и проявляет один Personal Tag.

```text
наблюдаемые факты первой вылазки
  -> Normal Threshold подтверждает возвращение
  -> Chronicle: First Return
  -> один eligible light либо situational tag
```

Тег берётся не из дерева и не из моральной оценки. Он обязан быть телесным, практическим либо материально закреплённым результатом вылазки. Система не превращает «был осторожен», «выбрал добро» или «успешно вышел» в универсальный бонус.

Точный способ разрешения случая, когда вылазка не создала очевидного сильного воздействия, остаётся предметом контентного прототипа. Он должен выдавать лёгкое читаемое проявление, а не скрытый случайный power-roll и не повод намеренно убить Ward ради следующей попытки.

## 4. Origin Foundling

Спасённый Foundling раскрывается с:

- одной Chronicle-записью происхождения;
- одним обычным `source_kind:: origin` Personal Tag;
- одной отдельной Origin Continuation по правилам Quest Engine.

Origin занимает одно из трёх механических мест и может иметь форму `light` либо `situational`. Он может быть редким по частоте, но не создаёт повышенный потенциал, дополнительный слот или отдельный силовой пул.

Спасение раскрывает уже прожитое свойство человека; оно не авторит его из качества rescue и не повышает rarity за срочность сцены.

## 5. Жизненный цикл

```text
Pawn life-run
  -> Chronicle накапливает факты
  -> отдельные eligible события создают Personal Tags
  -> Normal Threshold закрепляет вынесенный результат
  -> KIA заканчивает личную механическую конфигурацию
  -> аккаунт сохраняет только внешнее наследие
```

Breakline сохраняет ту же живую Пешку, Chronicle и активные теги ценой текущей материальной ставки. Смерть не создаёт компенсационный tag-roll следующему человеку.

Аккаунт может сохранить схрон, рецепты, известные источники, исполненные контракты, отношения Очагов и мировые последствия. Телесное или освоенное свойство умершей Пешки не наследуется как бонус.

## 6. Защита от фарма

- повтор действия не образует XP-рецепт;
- First Return происходит один раз на человека;
- tag source должен быть конкретным событием, практикой, воздействием либо реликтовой процедурой;
- KIA, handoff и увольнение не обновляют пул кандидатов и не уменьшают цену следующего источника;
- Chronicle может расти без создания новых механических правил;
- при трёх Personal Tags новое событие остаётся Chronicle, временным состоянием или Cargo, если не существует физического способа освободить носитель;
- rarity описывает частоту источника, а не автоматическое превосходство.

## 7. Игроковое чтение

После вылазки Стол показывает причинную цепочку раздельно:

```text
что произошло
  -> что осталось только памятью или отношением
  -> какое свойство тела / практики действительно возникло
  -> точное правило тега и его число либо trigger
  -> какая отдельная цель вернулась через Quest Engine
```

Игрок не должен принимать личный квест за пассивный бонус, а механический тег — за декоративный абзац биографии.
