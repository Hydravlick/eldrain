# Адаптивный арсенал и профильные ёмкости

Оружие и модули Термоса используют один источник — `Race × Practice` (`Race × Spec` в схеме) и постоянные теги, — но разные правила допуска. Оружие имеет **уровень владения**; модульная сборка имеет **профильные ёмкости**. Эти числа нельзя складывать или показывать одной шкалой.

## 1. Адаптивный Арсенал (Weapon Gating)
У персонажа нет навыка "Владение всем". Есть **Белый Список (Whitelist)** — `Allowed_Weapons`, который формируется при генерации.

$$Allowed = (RaceList \cup SpecList) - RaceBanned$$

1.  **Race List:** То, что существо может держать физически (Ёж может держать тяжелый фрейм, Крыса — нет без обхода).
2.  **Spec List:** То, чему существо обучено своей практикой (Авангард умеет стабилизировать магострел под давлением).
3.  **Race Banned:** То, что существо НЕ может использовать, даже если практика позволяет (например, `[Small Frame]` запрещает `[Heavy]` оружие).

**MVP-источник правды:** пока RaceList/SpecList не вынесены в отдельные поля рас и практик, итоговый `Allowed Arsenal` читается из `Registry_Combos`. Это сохраняет текущую систему, где P/Q/E и арсенал уже описаны вместе как результат смешения расы и практики.

Каждая утверждённая ячейка обязана иметь не один «правильный» предмет, а несколько конкретных `weapon_instance` из страниц фреймов: 2–3 экземпляра на `prof:: 2`, 1–2 экземпляра на `prof:: 1`, без строк `prof:: 0`. Базовые ёмкости шести семейств модулей и 2–4 доктрины снаряжения остаются отдельным слоем. Доктрины должны включать доступную, сбалансированную и ситуационную сборку; поздняя теговая/трейтовая специализация добавляется только при необходимости.

## 2. Владение оружием (Proficiency Ranks)
Вместо жесткого запрета "Нельзя", используется система **Soft Gating** (Мягкое ограничение) через уровни владения. Это не Tier конструкции: Tier предмета определяет допустимую нагрузку и устойчивость рабочего цикла, а уровень владения — доступные техники обращения с оружием.

**Формула:** `Базовое владение (Раса/Практика) + Теги (Чипы/Мутации) = Итоговый уровень`.

| Уровень владения | Статус | Эффект в игре |
| :--- | :--- | :--- |
| **0** | **Чужак** | **Блокировка.** Персонаж не может экипировать предмет. |
| **1** | **Новичок** | **Штраф.** Можно экипировать, но использование неэффективно (штрафы к анимации/отдаче). |
| **2** | **Профи** | **Норма.** Базовое состояние для родной практики. Оружие работает на 100%. |
| **3+** | **Мастер** | **Гейт открыт.** Оружие раскрывает мастерское свойство фрейма: технику, стабильность, cancel, снижение экспозиции или новое окно исполнения. |


## 3. Расширение Арсенала
Игрок может обойти ограничения и повысить уровень владения, устанавливая **Теги** (Импланты/Мутации).
* *Пример:* Тег `[Servo-Arm]` позволяет Крысе игнорировать запрет на `[Heavy]` оружие.
* *Пример:* Тег `[Trench Reflex]` дает `+1 к Магострелу`, превращая Новичка в Профи.

## 4. Фрейм как временное обязательство

Оружие не добавляет новый активный вектор в Combat Profile. Вектор фрейма (`frame_vector`) является временной надстройкой обязательства, а не частью архетипа.

```text
final_prof = combo_prof + tag_prof_delta + temporary_modifiers
if final_prof >= mastery_gate:
    unlock mastery_unlock for the current frame
```

`frame_vector` используется только для moment/exposure-аналитики:

```text
base_vector = кто Пешка по телу и практике
frame_vector = чем она временно выставилась во время действия
```

Фрейм становится активной экспозицией только в фазах `activates_on`: замах, прицеливание, удержание заряда, выстрел, блок или recovery. Оружие в рюкзаке и второй quick-slot не меняют слабости и не участвуют в Двойном Парадоксе.

Рекомендуемый MVP-контракт фреймов:

| Семейство | Типичный `frame_vector` | Что даёт | Чем платит |
|:---|:---|:---|:---|
| `blade` | `shadow` | вход, добор, мягкие зоны | броня, фронтальный обмен, риск входа |
| `blunt` | `kinetics` | guard break, stagger, пролом | замах, шум, recovery |
| `polearm` | `kinetics` | дистанция, удержание входа | мёртвая зона, фланг, вес |
| `arcanegun` | `ballistics` | линия давления, stagger, weakspot | шум, Heat, open line, interrupt |
| `catalyst` | `aether` | reality burn, эфирное окно | Dissonance, backlash, прерывание |
| `shield` | `kinetics` | покрытие, advance, парирование | фланг, вес, низкая мобильность |

Высокий proficiency улучшает работу с этими ценами, но не превращает фрейм в третий базовый вектор. Базовая общая слабость `Race × Practice` сохраняется всегда.

## 5. Профильные ёмкости Термоса

Ёмкость показывает, какую суммарную сложность модулей одного семейства Пешка способна обслуживать в одной вшитой сборке.

```text
FinalModuleCapacity(family) = max(0,
  ComboModuleCapacity(family)
  + sum(StableTagModuleCapacityDelta(family)))

UsedModuleCapacity(family) =
  sum(InstalledModuleCost(family))
```

Сборка допустима, если `UsedModuleCapacity <= FinalModuleCapacity` отдельно для каждого семейства. Очки не тратятся навсегда: они резервируются установленными модулями до следующей работы мастера.

| Семейство | Что обслуживает |
|---|---|
| `plate` | жёсткие пластины и распределение удара |
| `optic` | сенсоры, прицелы и информационные выводы |
| `seal` | герметизацию, фильтрацию и защиту среды |
| `conduit` | энергетические ветви, Heat и Backlash |
| `rig` | обвязку, инструменты и специализированный Ready Access |
| `weave` | мягкие слои мобильности, скрытности и кантрипа |

MVP-источник базовых значений — `Registry_Combos`:

```markdown
[module_capacity:: plate 2, optic 1, rig 1]
```

Теги используют отдельное поле и не смешиваются с оружейным `prof_delta`:

```markdown
[module_capacity_delta:: weave +1, plate -1]
```

В расчёте монтажа участвуют только постоянные значения. Временные эффекты, баффы и состояния рейда не позволяют установить модуль. Ёмкость не добавляет слоты, не меняет физическую позицию и не отменяет цену веса, Диссонанса или Gate Check.

Гибридный модуль резервирует ёмкость каждого семейства отдельно. Например, `[module_cost:: plate 2, conduit 1]` требует одновременно `plate >= 2` и `conduit >= 1`, хотя может занимать один физический слот.

## 6. Монтаж и мягкие ограничения

Оружие допускает мягкое владение уровня `1` прямо в рейде. Для модулей используется другой момент решения: мастер в Хабе либо подтверждает всю сборку, либо показывает конкретный дефицит ёмкости, слота, положения или посадки.

После монтажа сборка получает `stitched_locked`. В Аномалии модули нельзя менять, а найденные экземпляры остаются Cargo. Полный контракт: [[07_Gear_Inventory/Thermos_System|Термос]].
