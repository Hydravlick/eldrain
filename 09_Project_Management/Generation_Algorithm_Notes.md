#### Этап 1: 2D Топология и Физика (Floor Layout)

_Задача:_ Разместить комнаты (прямоугольники, круги, n-гоны), обработать коллизии и применить правила соседства (теги).

- **Вход:** Список комнат с тегами (напр., `[Кухня, Гостиная]`) и их желаемые связи.
    
- **Алгоритм:** Force-Directed Graph (Силовая укладка) + Rigid Body Dynamics.
    
    1. Представить комнаты как твердые тела.
        
    2. Применить силы притяжения для смежных по тегам комнат (пружины).
        
    3. Применить силы отталкивания для перекрывающихся тел (Collision Response).
        
    4. **Real-time Feature:** Когда пользователь тянет стену/комнату, она становится "кинематическим" объектом (бесконечная масса), расталкивая остальные.
        
- **Сложность:** $O(N^2)$ (где N — кол-во комнат).
    
    - _Оптимизация:_ С использованием Spatial Hashing или Quadtree сложность падает до $O(N \log N)$.
        
- **Результат:** Набор непересекающихся полигонов на 2D плоскости.
    

#### Этап 2: Триангуляция и Поиск Соседей (Adjacency Graph)

_Задача:_ Определить, какие комнаты касаются друг друга, чтобы применить «Приоритет стен».

- **Алгоритм:** Voronoi / Delaunay или простой Sweep-Line для поиска общих ребер.
    
    1. Для каждого ребра полигона пола ищем, есть ли ребро другого полигона в пределах $\epsilon$ (эпсилон, погрешность).
        
    2. Если нашли — создаем логическую связь `EdgePair {RoomA, RoomB}`.
        
- **Сложность:** $O(E \log E)$ (где E — общее число ребер всех полов).
    
- **Результат:** Граф смежности (Adjacency Graph), где ребра знают своих владельцев.
    

#### Этап 3: Генерация Объема (Straight Skeleton)

_Задача:_ Поднять стены и построить крышу.

- **Алгоритм:** Взвешенный Straight Skeleton (WSS).
    
    1. Для каждого полигона пола запускаем процесс «сжатия» (shrinking).
        
    2. Вершины движутся внутрь по биссектрисам углов. Скорость движения зависит от наклона крыши (параметризация).
        
    3. События (Edge Collapse, Split Event) формируют хребты крыши.
        
    4. Стены — это вертикальная экструзия от ребра пола до высоты, определенной скелетом.
        
- **Сложность:** $O(V^2)$ или $O(V \log V)$ в лучших реализациях (где V — вершины конкретной комнаты).
    
    - _Важно:_ Это самая тяжелая часть для реал-тайма.
        
- **Результат:** 3D меш крыши и базовые меши стен.
    

#### Этап 4: Разрешение Материалов Стен (Material Resolver)

_Задача:_ Если стена общая, выбрать материал по приоритету.

- **Вход:** Список `EdgePair` из Этапа 2.
    
- **Логика:**
    
    Python
    
    ```
    RuleMap = { "Stone": 10, "Wood": 5, "Glass": 1 }
    foreach pair in EdgePairs:
       MatA = pair.RoomA.Material
       MatB = pair.RoomB.Material
       FinalMat = (RuleMap[MatA] > RuleMap[MatB]) ? MatA : MatB
       pair.GenerateWall(FinalMat)
    ```
    
- **Сложность:** $O(E)$ (линейно от количества общих стен).
    
- **Результат:** Финальные стены с правильными материалами (камень перекрыл стекло).
    

#### Этап 5: Рекурсия для Башен (Vertical Sockets)

_Задача:_ Сгенерировать этажи N+1.

- **Алгоритм:**
    
    1. На крыше (результат Этапа 3) находим плоские участки или специальные точки-сокетов.
        
    2. Проверяем нормаль (должна смотреть вверх) и площадь.
        
    3. Если условия выполнены — создаем новую «комнату» (Floor) на высоте $H$.
        
    4. Вызываем **Этап 3** для новой комнаты.
        
- **Сложность:** $O(S)$ (где S — количество сокетов).
