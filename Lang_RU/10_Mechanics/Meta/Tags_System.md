# Система: Теги (The Tag Engine)

## 1. Философия
В Элдрейне всё состоит из Тегов. Персонаж — это просто контейнер для коллекции тегов.
* **Формула Сущности:** `Entity = Base_Tags + Gear_Tags + Status_Tags`.

## 2. Структура Тега
Каждый тег имеет скрытые параметры, влияющие на характеристики:

| Тип Тега | Пример | Эффект (Modifiers) | Приоритет |
| :--- | :--- | :--- | :---: |
| **Origin** (Раса) | `[Tag: Lazur]` | +2 AGI, +NightVision, -Strength | 1 (Base) |
| **Vocation** (Спец) | `[Tag: Alchemist]` | +Skill: Transmute, +1 Potions Slot | 2 (Base) |
| **Trait** (Черта) | `[Tag: Paranoid]` | +Detection Radius, +Stress Gain | 3 (Permanent) |
| **Gear** (Снаряжение) | `[Tag: Heavy_Plate]` | +Armor, -Movement Speed, +Noise | 4 (Item) |
| **Condition** (Сост) | `[Tag: Broken_Leg]` | -50% Move Speed, Cannot Sprint | 5 (Temporary) |

## 3. Взаимодействие Тегов (Tag Alchemy)
Теги могут блокировать или усиливать друг друга (Синергия).

### Конфликты (Blocking)
Если персонаж получает тег, несовместимый с существующим, новый тег либо не применяется, либо заменяет старый (зависит от источника).
* *Пример:* `[Tag: Heavy_Plate]` блокирует `[Skill: Acrobatics]`.

### Эволюция (Synergy)
Наличие двух определенных тегов может породить третий — **Скрытый Тег**.
* `[Tag: Pyromancer]` + `[Tag: Oil_Soaked]` = `[Auto-Tag: Walking_Bomb]` (Игрок взрывается при получении огненного урона).

## 4. Влияние на Геймплей
1.  **Навыки:** Игрок получает активные скиллы ТОЛЬКО от тегов. Нет дерева прокачки, есть поиск новых тегов (артефактов/книг).
2.  **Диалоги:** NPC реагируют на теги.
    * *Страж Традиций:* "Я не говорю с `[Tag: Occultist]`. Уходи."
3.  **Аномалии:** Некоторые аномалии наносят урон только носителям конкретных тегов (напр., "Ржавый Туман" разъедает `[Tag: Metal_Armor]`).

#### [Раздел: Создание Персонажа / Конструктор]

**Концепция: Сборка Сущности** Вместо классического выбора класса из списка, игрок попадает в **Лабораторию Сущностей (Constructor UI)**. Процесс создания визуализирован как «сборка» души из доступных космических отпечатков.

1. **Выбор Происхождения (Race/Origin):**
    
    - Игрок выбирает расу (например, _Лазур_ или _Человек_).
        
    - **Визуализация:** Модель персонажа меняется. Справа в интерфейсе обновляются базовые статы (Сила, Ловкость, Интеллект).
        
    - **Влияние:** Раса задает _потолок_ характеристик и пассивные свойства (например, _Ночное зрение_).
        
2. **Выбор Призвания (Class/Vocation):**
    
    - Игрок выбирает «Профессию» (Тег: _Алхимик_, _Наемник_, _Вор_).
        
    - **Динамический интерфейс:** При выборе тега, в слоте «Активный Навык» мгновенно появляется соответствующая карта способности.
        
        - _Пример:_ Выбрали `[Tag: Alchemist]` —> появилась иконка навыка `[Skill: Potion Throw]`.
            
        - _Пример:_ Выбрали `[Tag: Knight]` —> появилась иконка `[Skill: Shield Bash]`.
            
3. **Кастомизация (Traits):**
    
    - Игрок может взять дополнительные черты (Traits) для баланса. Некоторые черты негативные, но дают бонусы к статам.
        

**Итог:** Игрок не просто выбирает "Воина", он собирает билд: _«Лазур (Race) с навыками Вора (Class)»_.
