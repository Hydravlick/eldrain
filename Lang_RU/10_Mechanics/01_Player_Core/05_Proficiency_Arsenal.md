# Адаптивный Арсенал и Мастерство

## 1. Адаптивный Арсенал (Weapon Gating)
У персонажа нет навыка "Владение всем". Есть **Белый Список (Whitelist)** — `Allowed_Weapons`, который формируется при генерации.

$$Allowed = (RaceList \cup SpecList) - RaceBanned$$

1.  **Race List:** То, что существо может держать физически (Ёж может держать бревно, Крыса — нет).
2.  **Spec List:** То, чему существо обучено (Солдат умеет стрелять из винтовки).
3.  **Race Banned:** То, что существо НЕ может использовать, даже если Класс позволяет (например, `[Small Frame]` запрещает `[Heavy]` оружие).

## 2. Система Мастерства (Proficiency Tiers)
Вместо жесткого запрета "Нельзя", используется система **Soft Gating** (Мягкое ограничение) через уровни владения.

**Формула:** `Базовое владение (Раса/Класс) + Теги (Чипы/Мутации) = Итоговый уровень`.

| Уровень (Tier) | Статус | Эффект в игре |
| :--- | :--- | :--- |
| **0** | **Чужак** | **Блокировка.** Персонаж не может экипировать предмет. |
| **1** | **Новичок** | **Штраф.** Можно экипировать, но использование неэффективно (штрафы к анимации/отдаче). |
| **2** | **Профи** | **Норма.** Базовое состояние для "родного" класса. Оружие работает на 100%. |
| **3+** | **Мастер** | **Бонус.** Оружие становится продолжением тела (бонусы к статам). |


## 3. Расширение Арсенала
Игрок может обойти ограничения и повысить уровень владения, устанавливая **Теги** (Импланты/Мутации).
* *Пример:* Тег `[Servo-Arm]` позволяет Крысе игнорировать запрет на `[Heavy]` оружие.
* *Пример:* Тег `[Veteran Soldier]` дает `+1 к Магострелу`, превращая Новичка в Профи.
